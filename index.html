01 — 演算法介紹

什麼是
LoRA？

LoRA（低秩適應）是由 Microsoft Research 提出的「參數高效微調（PEFT）」技術，專為大型語言模型如 GPT-3 與 Stable Diffusion 等基礎模型設計。

其核心思想源自一個優雅的數學假設：大型預訓練模型在適應新任務時，所需的權重變化量存在於一個極低維度的子空間中。

「過度參數化的模型在適應特定任務時，其權重更新矩陣具有『低內在秩（Low Intrinsic Rank）』的特性。」

02 — 問題背景

它解決了
什麼難題？

  • 01
    儲存成本過高
    全量微調為每個下游任務產生一份與原模型同等大小的權重，難以在生產環境中部署。
  • 02
    Adapter 的推論延遲
    Adapter 以串聯方式插入額外網路層，在小批次線上推論場景下造成 20–30% 以上的延遲。
  • 03
    Prefix-tuning 的限制
    在輸入端加入前綴提示字元，壓縮模型可處理的有效序列長度，且效能不夠穩定。
03 — 核心機制

數學上的
精妙設計

LoRA 不改動原有的預訓練權重，而是在旁路引入兩個極小的矩陣，以最低成本實現任務適應。

Forward Pass 前向傳播計算公式
h  =  W₀x  +  ΔWx  =  W₀x  +  BAx
原始預訓練權重 W₀ 保持凍結，僅訓練低秩分解矩陣 B 與 A
W₀（凍結）
+
B（d × r）
A（r × k）
=
ΔW = BA

若原矩陣維度為 d × k，LoRA 引入：

  • A ∈ ℝʳˣᵏ以隨機高斯分佈初始化
  • B ∈ ℝᵈˣʳ初始化為全零，確保訓練起始時 ΔW = 0
  • r ≪ d,k秩 r 可低至 1 或 2，遠小於原始維度

推論時直接合併 W₀ + BA，無任何額外計算層。

01凍結原權重

預訓練模型的所有權重矩陣 W₀ 在訓練過程中完全凍結，不計算梯度，無需消耗對應記憶體。

02旁路低秩矩陣

在凍結層旁並聯引入 B、A 兩個低秩矩陣。其乘積 ΔW = BA 代表針對當前任務的「增量知識」。

03合併推論

訓練完成後，將 BA 直接加入原權重 W₀，得到 W = W₀ + BA，部署時推論延遲為零。

06 — 程式碼展示

從理論到
實作

LoRA 的核心邏輯不到 40 行 Python 即可實現。以下展示三種場景的偽代碼與實作參考。

import torch
import torch.nn as nn

class LoRALayer(nn.Module):
    """單一 LoRA 旁路模組 (ΔW = B @ A)"""
    def __init__(self, in_dim, out_dim, rank=4, alpha=1.0):
        super().__init__()
        self.rank  = rank
        self.scale = alpha / rank          # 縮放係數

        # A: 隨機高斯初始化；B: 全零初始化
        self.A = nn.Parameter(torch.randn(rank, in_dim) * 0.01)
        self.B = nn.Parameter(torch.zeros(out_dim, rank))

    def forward(self, x):
        # ΔWx = (B @ A)x，乘上縮放係數
        return (x @ self.A.T @ self.B.T) * self.scale


class LoRALinear(nn.Module):
    """將原 Linear 層包裹為 LoRA 版本"""
    def __init__(self, linear: nn.Linear, rank=4):
        super().__init__()
        self.linear = linear
        self.lora   = LoRALayer(linear.in_features,
                                  linear.out_features, rank)
        # 凍結原始權重
        for p in self.linear.parameters():
            p.requires_grad = False

    def forward(self, x):
        # h = W₀x + ΔWx
        return self.linear(x) + self.lora(x)

    def merge_weights(self):
        """推論前合併，消除額外計算"""
        delta_W = (self.lora.B @ self.lora.A) * self.lora.scale
        self.linear.weight.data += delta_W
# ── 訓練流程：僅更新 LoRA 參數 ──

def apply_lora_to_model(model, rank=4, target_modules=("q_proj", "v_proj")):
    """將指定 Attention 矩陣替換為 LoRA 版本"""
    for name, module in model.named_modules():
        if any(t in name for t in target_modules):
            parent, child_name = get_parent(model, name)
            original = getattr(parent, child_name)
            setattr(parent, child_name, LoRALinear(original, rank=rank))
    return model


# 套用 LoRA
model = apply_lora_to_model(pretrained_model, rank=4)

# 只優化 LoRA 參數（A, B 矩陣）
lora_params = [p for n, p in model.named_parameters()
               if "lora" in n and p.requires_grad]

optimizer = torch.optim.AdamW(lora_params, lr=3e-4)

# 訓練迴圈
for epoch in range(num_epochs):
    for batch in dataloader:
        loss = model(**batch).loss
        loss.backward()       # 只有 LoRA 參數有梯度
        optimizer.step()
        optimizer.zero_grad()

# 訓練完成：合併權重，準備零延遲部署
for module in model.modules():
    if isinstance(module, LoRALinear):
        module.merge_weights()
# ── 使用 HuggingFace PEFT 套件（推薦生產方式）──

Vaswani et al. 發表《Attention Is All You Need》，奠定現代大型語言模型的基礎架構，也埋下了日後 LoRA 需要解決的微調成本問題的種子。

基礎架構
2020
GPT-3 發布，微調成本危機浮現

OpenAI 發布擁有 1750 億參數的 GPT-3。全量微調一次的成本高達數百萬美元，且每個任務都需儲存完整副本——業界迫切需要更有效率的微調方法。

規模化挑戰
2021.06
LoRA 論文在 arXiv 發布（v1）

Hu et al. 來自 Microsoft Research 發表 arXiv:2106.09685。核心創新：以兩個低秩矩陣 B、A 的乘積代替全量權重更新，訓練參數減少 10,000 倍。

論文在第十屆國際學習表徵大會（ICLR 2022）正式發表，引發學術界廣泛關注，迅速成為 PEFT 領域引用最高的論文之一。

08 — 深層洞見

低秩矩陣
學到了什麼？

論文第 6–7 章透過數學分析揭示了 LoRA 真正運作的本質，遠不只是參數壓縮。

最佳秩 r 的奇效

在許多下游任務中，秩 r = 1 或 r = 2 就已足夠，與 r = 64 的效能幾乎相當。r = 8 與 r = 64 所學到的「最重要頂部特徵方向」高度重疊——多餘的 r 只是在學習累積的隨機雜訊。

r = 188.2
r = 489.8
r = 6490.1

特徵放大效應

ΔW 並非在模型中學習全新的隨機知識方向，而是找出了預訓練權重 W 中「已經學會、但針對當前任務尚未被充分強調」的特徵方向，並將其大幅度放大。

在 r = 4 的實驗中，特定任務相關特徵被放大了超過 21.5 倍。它不是在學習新知識，而是在「調焦」。

應該適應哪些矩陣？

給定相同的參數預算（18M），同時適應查詢矩陣 W_q 與鍵值矩陣 W_v，明顯優於將所有預算只集中在單一矩陣。廣泛覆蓋多個權重矩陣，比深入單點更加重要。

切換任務的極致靈活

每個任務對應的 ΔW 模組可能只有幾 MB，切換任務時只需抽換這兩個小矩陣。以 GPT-3 175B 為例，實現了「一個基礎模型，無限任務版本」的工業化部署範式。
